# Block Party 🎉

A gentle block-puzzle game for very young players. Drag the coloured pieces onto
the board and fill a whole row or column to make it sparkle and clear. Built for
Isabelle (age 4), modelled on the "Block Puzzle" app she loves.

**Play it:** https://rvenning.github.io/block-party/

## Features
- 🧩 Classic block puzzle: drag pieces onto an 8×8 grid, clear full rows and columns
- 💥 Combo bonuses when one placement clears several lines at once, plus a confetti **Super Clear** for emptying the whole board
- 🪙 Coins for every line cleared; ⭐ your best score is remembered
- 🧸 **No timer and no "Game Over"** — a full board is a celebration ("Wonderful!" / "New Best!"), never a fail screen
- 🤝 Kind by design for little hands: it always deals at least one piece that fits, invalid drops just hop back with a soft sound, the dragged piece floats above the finger, and lines that are about to clear light up before you commit
- 👨‍👩‍👧 Pick-a-name family profiles — no passwords; optional 4-digit PIN per profile with an admin override
- 🦄 Your profile avatar is your player; kid-friendly avatar set
- 🏆 Shared family leaderboard (best score)
- 🔊 All sound effects synthesized with WebAudio (no asset downloads)
- ☁️ Firebase Firestore sync (falls back to localStorage offline)
- 📱 iPad-first: big touch targets, drag-and-drop tuned for small hands; works with a mouse on desktop
- 📲 Installable PWA: "Add to Home Screen" button, offline play via service worker

## Scoring
- **+1** per cell placed
- **+10** per line cleared, with a **+15 × (lines − 1)** combo bonus for multi-line clears
- **Coins** = lines cleared; a full-board Super Clear adds **+50**
- Progress `{ best, coins, lines, games }` reconciles across devices by keeping the max of each field

## Built on gamekit
Profiles, PINs, storage/sync, the sound engine, base UI styles and the PWA kit come from
[rvenning/gamekit](https://github.com/rvenning/gamekit), vendored into `lib/`.
To pull in a newer kit version:

```powershell
# from the gamekit repo
node tools/sync-to-game.js "D:\OneDrive\Documents\Claude Code\block-party"
# then test, bump the CACHE version in sw.js, and commit
```

Everything game-specific — the canvas engine, the piece shapes, the grid/clear logic,
the jingles — lives in `index.html`.

## PWA files
- `manifest.json` — app identity: name, icons, standalone display, theme colours
- `sw.js` — service worker; precaches the app shell (including `lib/`), then network-first with cache fallback
- `icons/` — 192/512 PNG app icons plus a maskable variant, generated by `tools/make-icons.js` (no image tools needed — pure Node)

## Local development
No build step — plain HTML/JS. Serve the folder with any static server:

```powershell
npx http-server . -p 8098 -c-1   # http://localhost:8098
```

Regenerate the icons after changing the art:

```powershell
node tools/make-icons.js
```

Deploys are just pushes to `main` (GitHub Pages).

> **Canvas note:** the `<canvas>` needs an explicit CSS size (`#cv { width:100%; height:100% }`).
> A canvas is a replaced element, so `position:absolute; inset:0` alone won't stretch it —
> without a CSS size it renders at its attribute size (`W × devicePixelRatio`), which is 2–3×
> too large on retina screens. This is invisible on a desktop preview but overflows on any iPhone/iPad.

## Storage
Progress lives in `localStorage` (keys `bp_*`) and mirrors to Cloud Firestore
(collection `blockparty`, same Firebase project as WordVoyage and Chicken Cross).
The Firebase API key in `index.html` is public by design — it only identifies the
project; access is limited by Firestore security rules, and the key is
referrer-restricted to the game's domains and locked to the Firestore API.
